--------------------------

Version 2.2.2 (2026-07-09)
--------------------------
- [UI/UX] Added an inline glowing green status dot to the left of the "Connected" / "Connected (BG)" text inside the connection status badge for enhanced visual feedback.

Version 2.2.1 (2026-07-09)
--------------------------
- [Revert] Reverted renaming format back to using `_Part_` keyword (`<prefix>_Part_<index>.<ext>`) to ensure downstream workflow compatibility and prevent renaming failures.

Version 2.2.0 (2026-07-09)
--------------------------
- [Feature] Updated file renaming template format to output `<prefix>_<index>.<ext>` (e.g. `Hook_001.mp3`) instead of `<prefix>_Part_<index>.<ext>` by removing the redundant `_Part` keyword in `background.js`.

Version 2.1.9 (2026-07-09)
--------------------------
- [UI/UX] Added a beautiful, subtle breathing green glow animation (`btnSuccessGlow`) to the Completed button state to emphasize success states visually.

Version 2.1.8 (2026-07-09)
--------------------------
- [UI/UX] Constrained credits sync requests by introducing a `shouldRequestCredits` parameter inside `checkTabStatus()` in `popup.js`, ensuring that typing in the text area or changing configurations does not trigger duplicate background click-scans.
- [UI/UX] Restricted background click-scans to strictly fire ONLY on extension startup/open, page connection, page reload, and batch generation completion.

Version 2.1.7 (2026-07-09)
--------------------------
- [UI/UX] Removed the red-blue breathing gradient animation (`warning-active`) from the "Start Automation" button to keep the interface clean; the warnings display card is sufficient.
- [UI/UX] Added an active listener in the background worker script (`background.js`) to capture `creditsUpdate` events when the popup is closed, storing the balance securely in local storage so that reloading the page updates the balance even if the Side Panel is closed.

Version 2.1.6 (2026-07-09)
--------------------------
- [UI/UX] Made the background programmatic credits click-scan completely invisible by injecting a CSS layout style rule that hides poppers (`opacity: 0 !important; transform: scale(0) !important;`) while the `.fish-automator-scanning` class is active on the HTML document during the 150ms scan lifecycle.
- [UI/UX] Constrained programmatic click-scanning triggers to ONLY run when the automation queue is NOT processing chunks (e.g. at page load, popup connection, and batch completion), preventing dropdown flashing/flickering during generation.

Version 2.1.5 (2026-07-09)
--------------------------
- [UI/UX] Enhanced trigger simulation to specify primary mouse left-click button parameters (`button: 0`, `buttons: 1`) inside the simulated PointerEvent and MouseEvent event dispatches, resolving triggers on strict React UI targets.

Version 2.1.4 (2026-07-09)
--------------------------
- [UI/UX] Added a fallback `MutationObserver` on `document.body` that automatically triggers a credit sync as soon as the profile dropdown menu is rendered or mutated on the webpage DOM, ensuring real-time balance tracking.
- [UI/UX] Switched the dropdown trigger simulation to dispatch a comprehensive sequence of `PointerEvent` and `MouseEvent` (pointerdown, mousedown, pointerup, mouseup, click) to satisfy React / Radix UI event listeners which trigger on mouse down.
- [Bug Fix] Refactored `getProfileTrigger()` to crawl up the DOM hierarchy starting from the avatar image and match the exact outer dropdown wrapper element.

Version 2.1.3 (2026-07-09)
--------------------------
- [UI/UX] Added an invisible programmatic background scan routine (`autoScanCredits`) that automatically triggers on page connection or page refresh. This clicks the profile avatar, queries the dynamically loaded dropdown structure, updates the credits balance in storage, and closes it again immediately (in less than 150ms) to sync the credits fully automatically without the user having to manually click the profile menu.
- [UI/UX] Enhanced span search selectors to verify `title` attributes in addition to raw text content when parsing credit fractions in the DOM.

Version 2.1.2 (2026-07-09)
--------------------------
- [Bug Fix] Resolved issue where the credits badge was hidden by fixing `updateCreditsDisplay()` which overrode visibility settings by setting `display = 'none'` when `currentCredits` was null. The credits badge visibility is now solely controlled by the connection state tracker (`checkTabStatus()`), making the badge always display as `Credits: --` when connected until a balance sync succeeds.

Version 2.1.1 (2026-07-09)
--------------------------
- [UI/UX] Optimized `checkCredits` selector logic to directly match the HTML pattern of the team-switcher credit fractions (`span[class*="team-switcher"]` or matching `/^\d+[\d,]*\/\d+[\d,]*$/` format).
- [UI/UX] Configured the Credits status badge to remain visible as `Credits: --` upon tab connection and added an interactive click listener that guides the user on how to manually trigger a sync if needed.

Version 2.1.0 (2026-07-09)
--------------------------
- [UI/UX] Added a discrete Credits display badge at the top of the popup (`Credits: 2000`). If the current batch's total character count exceeds the remaining credit balance, the credits badge breathes fast and glows red, and the "Start Automation" button plays a smooth "blue-to-red and red-to-blue" transition animation.
- [UI/UX] Implemented a green "Completed!" success state. When a script finishes successfully, the status dots remain green, and the button transitions to a green "Completed!" state. The button reverts to "Start Automation" and dots turn grey only when clicked, or when the user makes changes to inputs, uploads, parameters, or chunk items.
- [Bug Fix] Trigger a native browser `alert()` pop-up on credit errors so that the user is immediately notified even if the Side Panel is minimized or out of focus.
- [Bug Fix] If generation is blocked by the website's plan upgrade/limit modal, the extension stops immediately and leaves the modal open for the user to see, rather than closing it automatically.

Version 2.0.5 (2026-07-09)
--------------------------
- [Bug Fix] Aligned the automation progress indicator percentage in `popup.js` to increment based on the count of *previously completed* chunks. The progress bar now correctly stays visible at less than 100% (e.g., 67% on the final chunk of 3) during the final chunk's generation and download.
- [Bug Fix] Added a 5-second completion delay *after* the final download button is triggered before sending the 100% completion message, ensuring the background script has finished intercepting and renaming the last file before the UI resets.

Version 2.0.4 (2026-07-09)
--------------------------
- [Bug Fix] Preserved renaming configuration in local storage when automation completes by removing the automatic storage wiping instruction in `resetUI` (popup.js). The renaming configuration stays active continuously so that the last file (regardless of download delays) is renamed correctly. The state is only cleared when the user clicks the "Stop" button, the "Dustbin/Clear" workspace button, or starts a new script.

Version 2.0.3 (2026-07-09)
--------------------------
- [Bug Fix] Delayed the final progress completion message along with the 10-second `clearPrefix` timeout inside `content.js`. This prevents `popup.js` from calling `resetUI()` and prematurely clearing the storage before the last download can start and resolve its filename.
- [Feature] Implemented a low credits warning banner in the popup interface that automatically alerts the user if remaining account credits fall below 500.
- [Bug Fix] Implemented dynamic page-level error scans for credit exhaust warnings ("insufficient credits", "limit reached", etc.) on fish.audio to halt the execution immediately with a clear description if generation is blocked.

Version 2.0.2 (2026-07-09)
--------------------------
- [Bug Fix] Resolved race condition on last chunk download renaming by delaying `clearPrefix` by 10 seconds inside `content.js` when execution completes. This ensures that the last download process has fully initialized and had its filename determined before the background worker cleans the active prefix state.

Version 2.0.1 (2026-07-09)
--------------------------
- [Bug Fix] Resolved sporadic renaming failures on subsequent chunks. Refactored the background download listener to track state variables (`downloadPrefix`, `downloadCounter`, and `automationActive`) in synchronous local service worker memory, allowing instant, non-blocking suggestions that bypass Chrome's filename determination timeouts.

Version 2.0.0 (2026-07-09)
--------------------------
- [Bug Fix] Resolved double-download bug by refactoring the pointer/mouse event simulation inside `content.js` to dispatch a single click event to the target element (removing the duplicate synthetic click dispatch).
- [UI/UX] Made "Auto-Rename Downloads" turned ON (checked) by default so renaming functions immediately out of the box.
- [Bug Fix] Expanded file name interception in `background.js` to match any audio files downloaded while automation is active, bypassing URL/CDN domain checks.
- [Feature] Updated the workspace dustbin/clear button (🗑️) to reset all storage, file buffers, and download counters, completely cleaning up the browser local storage state.
